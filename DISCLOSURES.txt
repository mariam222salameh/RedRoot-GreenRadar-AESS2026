RedRoot — Green Radar Flood Detection
AESS Sustainability Hackathon 2026 · Challenge 2 · Final Phase

1. Software Libraries (used in the core simulation)
- numpy: Numerical arrays, FFT, signal generation.
- scipy: Cosine-distance anomaly scoring (scipy.spatial.distance.cosine).
- matplotlib: All six validation plots.
All three are open-source, standard scientific Python libraries; no proprietary or paid dependencies are used in redroot_simulation.py.

2. External Data / Standards Referenced
- IEA Emissions Factors Database, 2023 Edition: Egyptian grid carbon intensity constant (0.233 kg CO₂/kWh) used in the CDD metric.
No other external datasets are used — all radar returns, flood-intensity curves, and sensor signals in the simulation are synthetically generated within the script itself (deterministic, seed = 42), not sourced from a real dataset.

3. Platforms Used to Build Project Assets
- Lovable (lovable.app): Built the project website / live-demo dashboard (redroot-green-radar.lovable.app).

- Drafting and formatting the repository README.md.

Review note: An earlier AI-assisted document (working title: "RedRoot Green Radar — Sustainable Cognitive Radar Validation Report") was produced during drafting and found on review to contain hardware specifications, TinyML performance figures, and accuracy/precision statistics that do not correspond to anything implemented in the team's actual codebase. That document was not included in the final submission in its original form. It was revised into a "Design Feasibility & Literature Grounding Report" — explicitly reframed and relabeled as a forward-looking literature review supporting the project's future roadmap (on-device TinyML compression, physical hardware deployment), not as a report of measured or validated results. Every figure in that document is now labeled "Design Target" rather than "Assigned Value," and the document states plainly that it is not a validation report. The team's only validated, reproducible results are those in redroot_simulation.py's single 24-hour simulation.
